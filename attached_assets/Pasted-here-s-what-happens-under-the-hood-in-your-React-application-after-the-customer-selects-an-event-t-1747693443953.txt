Reactive Re-render & Conditional Logic: This state change triggers React to re-render components subscribed to it. Your main form orchestrator (QuotationWizard.tsx) sees the new selectedEventType and its internal logic kicks in:

The part of the UI that showed the event options might now be conditionally hidden.
The section designed to display the actual form steps becomes active.
Dynamic Form Configuration: QuotationWizard.tsx uses the selectedEventType to look up or generate the specific configuration for the "Wedding" form. This configuration is likely a JavaScript object detailing:

The specific React components for each step (e.g., WeddingDetailsStep.tsx, WeddingMenuStep.tsx).
Any default values for react-hook-form fields relevant to weddings.
The Zod validation schema (or relevant parts of a larger schema) for the wedding form.
React Hook Form Adaptation: The instance of react-hook-form (likely initialized in QuotationWizard.tsx and provided via <FormProvider>) is now prepared for the wedding form:

It might be reset with the default values from the wedding form configuration.
The validation resolver will use the wedding-specific Zod schema.
Rendering the New Form Content: React efficiently updates the DOM. Instead of rendering the event selection UI, QuotationWizard.tsx now renders the first step component of the wedding form sequence (e.g., <WeddingDetailsStep />) within the existing page layout. The page header, footer, and overall shell remain identical.

Seamless Visual Transition: To the customer, this appears as if the content area of the page smoothly changed. There's no browser page reload. You might use subtle CSS transitions (like a quick fade or slide) on the outgoing event selection UI and the incoming form step to enhance this feeling of a fluid, single-page application.
