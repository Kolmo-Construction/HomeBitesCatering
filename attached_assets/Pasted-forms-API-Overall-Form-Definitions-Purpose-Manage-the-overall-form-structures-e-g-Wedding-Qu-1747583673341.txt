forms API (Overall Form Definitions)

Purpose: Manage the overall form structures (e.g., "Wedding Quote Form").
Endpoints:
POST /api/form-builder/forms
Action: Create a new form definition.
Request Body: { formKey, formTitle, description, version, status }.
Response: The created form object.
GET /api/form-builder/forms
Action: List all forms (with pagination/filtering by status).
Response: Array of form objects.
GET /api/form-builder/forms/:id (or /:formKey/:version)
Action: Get a specific form definition.
Response: Single form object.
PUT /api/form-builder/forms/:id
Action: Update a form definition (e.g., title, description, status). For versioning, updating a "published" form might create a new version.
Response: The updated form object.
DELETE /api/form-builder/forms/:id
Action: Delete a form definition (consider soft delete or restrictions if it has submissions).
Response: Success/failure message.
3. form_pages API (Pages within a Form)

Purpose: Manage the pages belonging to a specific form.
Endpoints (often nested under a form route):
POST /api/form-builder/forms/:formId/pages
Action: Create a new page within a form.
Request Body: { pageTitle, pageOrder, description }.
Response: The created page object.
GET /api/form-builder/forms/:formId/pages
Action: List all pages for a given form, ordered by pageOrder.
Response: Array of page objects.
PUT /api/form-builder/forms/:formId/pages/:pageId
Action: Update a page (title, order, description).
Response: The updated page object.
DELETE /api/form-builder/forms/:formId/pages/:pageId
Action: Delete a page (questions on it might cascade delete).
Response: Success/failure message.
POST /api/form-builder/forms/:formId/pages/reorder
Action: Batch update page orders for a form.
Request Body: Array of { pageId, newPageOrder }.
Response: Success/failure message.
4. form_page_questions API (Question Instances on a Page)

Purpose: Manage the instances of library questions placed on form pages, including their overrides.
Endpoints (nested under a page route):
POST /api/form-builder/pages/:pageId/questions
Action: Add a question from the question_library to a form_page.
Request Body: { libraryQuestionId, displayOrder, displayTextOverride, isRequiredOverride, isHiddenOverride, metadataOverrides, optionsOverrides }.
Response: The created form_page_questions instance object.
GET /api/form-builder/pages/:pageId/questions
Action: List all question instances for a page, ordered by displayOrder, and potentially joining with question_library to get default details.
Response: Array of question instance objects (with merged/resolved properties).
PUT /api/form-builder/pages/:pageId/questions/:questionInstanceId
Action: Update a question instance on a page (e.g., its overrides, order).
Response: The updated question instance object.
DELETE /api/form-builder/pages/:pageId/questions/:questionInstanceId
Action: Remove a question instance from a page.
Response: Success/failure message.
POST /api/form-builder/pages/:pageId/questions/reorder
Action: Batch update question instance orders for a page.
Request Body: Array of { questionInstanceId, newDisplayOrder }.
Response: Success/failure message.
5. form_rules & form_rule_targets API

Purpose: Manage conditional logic rules for a form.
Endpoints (nested under a form route):
POST /api/form-builder/forms/:formId/rules
Action: Create a new conditional rule.
Request Body: { triggerFormPageQuestionId, conditionType, conditionValue, actionType, ruleDescription, executionOrder, targets: [{ targetType, targetId }, ...] }. The backend would create the form_rules record and associated form_rule_targets records.
Response: The created rule object (with its targets).
GET /api/form-builder/forms/:formId/rules
Action: List all rules for a form (including their targets).
Response: Array of rule objects.
PUT /api/form-builder/forms/:formId/rules/:ruleId
Action: Update an existing rule (and its targets - might involve deleting old targets and creating new ones).
Response: The updated rule object.
DELETE /api/form-builder/forms/:formId/rules/:ruleId
Action: Delete a rule (targets would cascade delete).
Response: Success/failure message.
