Here is the breakdown of how to handle this.

1. The "ID" you need is the Thread ID
You are currently capturing message.getId(). This is unique to every single email. To group them into a timeline, you need the Gmail Thread ID.

Message ID: msg-123 (Specific email: "Re: Quote")

Thread ID: thread-abc (The whole conversation context)

When you send the threadId to your database, your app can simply query: "Show me all messages where thread_id = thread-abc."

2. The Logic Change (Handle Replies vs. Leads)
Currently, your script scores every email.

Problem: A customer replies saying "Yes, that works."

Result: Your script sees no keywords (catering, guests, food), scores it a 0, and ignores it.

The Fix: Check if the email thread already has your "Parsed" label.

If labeled: It is a reply to an existing lead. Auto-Pass (Skip scoring).

If NOT labeled: It is a stranger. Run Scoring (Check for keywords).

function processEmail(message, threadId, type) {
  try {
    const emailData = {
      gmailMessageId: message.getId(),
      gmailThreadId: threadId, // <--- CRITICAL FOR TIMELINE
      type: type, // 'new_lead' or 'timeline_reply'
      subject: message.getSubject(),
      from: message.getFrom(),
      to: message.getTo(),
      receivedDate: message.getDate().toISOString(),
      body: {
        plainText: message.getPlainBody(),
        htmlBody: message.getBody()
      }
    };
    
    // ... existing cleaning logic ...
    
    // Update your cleanEmailText call and API call as normal
    let cleanedText = cleanEmailText(emailData.body.plainText);
    const apiResponse = sendToReplitAPI(emailData, cleanedText);
    
    return { success: true, leadId: apiResponse.leadId };
    
  } catch (error) {
    return { success: false, error: error.message };
  }
