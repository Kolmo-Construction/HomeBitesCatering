Step 3: Configure Your Category Names and Item Assignments in Your CMS

This is where you define how each theme should look for the customer. You'll need to do this for every menu theme you want to display on the inquiry form.

Interface: Your "Home Bites CMS" menu editor (the interface shown in Screenshot 2025-05-30 at 4.26.32 PM.png).
For Each Menu Theme (e.g., "A Taste of Greece," "American BBQ"):
Define Categories: When editing a theme, ensure you create category groupings with the exact display names you want customers to see. For "A Taste of Greece," these would be "Mains," "Salads," "Pasta," "Salsas," "Sauces," "Sides," "Spreads."
Your CMS should allow you to set a "Display Title" (or similar field) for each of these category groupings. This is what will appear as the heading on the form after Step 2's code change.
It will also likely have an internal "Category Key" (e.g., mains, pasta_dishes). Keep these simple and consistent.
Assign Items: Carefully assign each of your individual menu items (which have their own general categories like "Main Course - Meat") to one of these theme-specific display categories you've just defined (e.g., assign "Papoutsakia" to your "Mains" category within the "A Taste of Greece" theme).
Step 4: Ensure Your CMS Saves This Detailed Structure to the Database

What happens: When you save a menu theme in your CMS, it must save this detailed category structure (including your desired display_title for each category, the category_key, and the list of available_item_ids for that category) into the items JSONB column of the public.menus table for that specific menu's record.
Verification (Conceptual): This step is mostly about trusting your CMS works correctly. If it doesn't save the display_title properly, or if it doesn't structure the categories array as expected by generateMenuData.js (which expects objects with category_key, display_title, and available_item_ids), then the generated JSON will be incorrect.
