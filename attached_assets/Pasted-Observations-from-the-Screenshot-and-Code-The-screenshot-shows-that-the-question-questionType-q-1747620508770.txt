The screenshot shows that the question.questionType || question.question_type (source 725) is being displayed (e.g., "matrix", "textbox").
It seems the question.displayText || question.default_text (source 724) is not being displayed for these items. Instead, the question type itself (like "matrix", "textbox") is appearing where the primary display text should be. This is the main visual discrepancy.
The helperText (source 726) is also not visible in the screenshot for these items, which might be expected if the displayText field is also missing or empty for those specific library questions.
Potential Causes for Incorrect Display:

Missing displayText or default_text in Library Question Data:

The most likely cause is that the actual question objects fetched from /api/form-builder/library-questions and stored in libraryQuestionsData.data (source 699) do not have the displayText or default_text properties populated for the questions shown ("matrix", "textbox", etc.).
The code question.displayText || question.default_text (source 724) tries to use displayText first, then falls back to default_text. If both are missing or empty for these specific library questions, the p tag would be empty or a falsy value, which might not render as expected or might be styled to be invisible if empty.
However, the screenshot shows the question type ("matrix", "textbox") in the place where the displayText should be. This suggests that question.displayText might actually contain the question type string instead of the intended question text.
Incorrect Data Mapping or Property Names:

When library questions are created or fetched, there might be an issue with how displayText (or defaultText / default_text) is being stored or retrieved.
The backend GET /api/form-builder/library-questions endpoint in formRoutes.ts (or questionLibraryRoutes.ts - there seem to be two files managing similar routes) should return objects where defaultText (or default_text in the database schema questionLibrary.defaultText (source in shared/form-schema.ts)) contains the actual question's text/label.
In clea/server/questionLibraryRoutes.ts, the createQuestionSql (source in server/questionLibraryRoutes.ts) and updateSql (source in server/questionLibraryRoutes.ts) correctly use default_text for insertion/update, and getQuestionWithDetails (source in server/questionLibraryRoutes.ts) selects *. So the data should be available from the DB.
In clea/server/formRoutes.ts, the create endpoint (source in server/formRoutes.ts) uses validatedData which comes from insertQuestionLibrarySchema.parse(req.body). The insertQuestionLibrarySchema (source in shared/form-schema.ts) is based on questionLibrary table which has defaultText. This seems correct.
Data Transformation Issue:

Somewhere between fetching from the database and rendering in the FormEditor.txt, the displayText or default_text property might be getting overwritten or lost for these specific questions.
