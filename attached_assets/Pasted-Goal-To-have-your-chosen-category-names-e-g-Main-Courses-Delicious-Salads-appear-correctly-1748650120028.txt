Step 1: Update How the Inquiry Form Displays Category Titles (A One-Time Code Adjustment)

What you're doing: You'll tell the wedding inquiry form to use the nice, descriptive "Display Title" for each category section, instead of just its internal system name or key.
File to modify: client/src/pages/wedding/ComprehensiveWeddingInquiry.tsx
Specific change:
Inside this file, find the part of the code that shows the menu items grouped by category when a customer has selected a theme (this is within the renderMenuSelectionStep function).
You'll see a line that creates the heading for each category, which currently looks something like: <h4 className="font-semibold text-lg mb-3 capitalize">{categoryKeyFromFile}</h4>
You need to change this line so it uses the title property from the category's details (this title property is sourced from the display_title you set in your database/CMS). Change it to: <h4 className="font-semibold text-lg mb-3">{categoryDetailsObject.title || categoryKeyFromFile}</h4>
This tells the form: "Try to use the nice title for the heading. If for some reason it's missing, just use the categoryKeyFromFile as a backup."
