API Endpoint to Handle Form Submissions



Objective: Create a POST endpoint to receive client-submitted form data, perform server-side validation against the specific form version, and store the submission and answers.

Endpoint Signature (Example): POST /api/forms/:formKey/versions/:versionNumber/submit

Key Backend Logic:Receive Request: Get formKey, versionNumber, and the submission payload.

Payload Structure: Expect answers mapped to form_page_questions.id (the instance ID). Also expect linking info like clientId, opportunityId (if applicable and known by the client).

Fetch Form Definition for Validation: Retrieve the exact form definition (for the submitted formKey and versionNumber) – you might reuse parts of the logic from the "Render Form" API or a common service function.

Server-Side Validation (Critical):Against the fetched form definition, validate all submitted answers:Required Fields: Check if all questions that should have been visible and are marked as required have answers. This requires re-evaluating conditional logic based on the submitted answers up to that point.

Data Types & Constraints: Validate answer values against the questionType and resolved metadata (e.g., min/max values for numbers, length for text, valid email, selection within allowed options, file type/size for uploads if you implement that type).

If validation fails, return a detailed 400 Bad Request response.

Store Data (Within a Database Transaction):Create a new record in form_submissions, setting formId, formVersion, status (e.g., 'completed'), submittedAt, and linking IDs (clientId, opportunityId, userId from session if applicable, rawLeadId).

For each answer provided, create a record in form_submission_answers, linking it to the form_submissions.id and the corresponding form_page_questions.id, storing the answerValue (JSONB).

Post-Submission Actions :Trigger notifications (e.g., email to admin, email to client).

Create tasks or update records in your CRM (e.g., update opportunity stage).

Return Response: Success message (e.g., 201 Created with the submission ID or the created submission object).
