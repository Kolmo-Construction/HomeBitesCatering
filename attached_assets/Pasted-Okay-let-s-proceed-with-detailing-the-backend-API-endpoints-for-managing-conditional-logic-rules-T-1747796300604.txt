Okay, let's proceed with detailing the backend API endpoints for managing conditional logic rules. These will be added to server/formBuilderRoutes.ts.

We'll use Drizzle ORM for database interactions, leveraging the schemas already defined in shared/form-schema.ts (specifically formRules and formRuleTargets).

File: server/formBuilderRoutes.ts

I will add the following route handlers to the existing router instance.

1. Create a new Rule

Endpoint: POST /api/form-builder/forms/:formId/rules
Purpose: Creates a new conditional rule associated with a form and triggered by a specific question instance within that form.
Request Body Schema (validation using Zod recommended):
JSON

{
  "triggerFormPageQuestionId": number, // ID of the formPageQuestion that triggers this rule
  "conditionType": "equals" | "not_equals" | "is_filled" | ... , // from conditionalLogicConditionTypeEnum
  "conditionValue": string | null, // Value for the condition (optional for some types like 'is_filled')
  "actionType": "show_question" | "hide_question" | ... , // from conditionalLogicActionTypeEnum
  "targets": [ // Array of targets
    {
      "targetType": "question" | "page", // from formRuleTargetTypeEnum
      "targetId": number // ID of the formPageQuestion or formPage
    }
  ],
  "ruleDescription": string | null, // Optional
  "executionOrder": number | null // Optional, defaults to 0 or next available
}
Logic:
Parse and validate formId from req.params.
Parse and validate the request body against the schema above.
Use db.transaction(async (tx) => { ... }) for atomicity.
Insert into formSchema.formRules table with the provided data, linking it to formId and triggerFormPageQuestionId.
Get the id of the newly created rule.
Iterate through the targets array from the request body. For each target:
Insert a new record into formSchema.formRuleTargets, linking it to the new ruleId, and setting targetType and targetId.
Fetch the newly created rule along with its targets to return in the response.
Response:
Success (201): The newly created rule object, including its id and an array of its targets.
Error (400/500): Error message.
2. Get All Rules for a Specific Form

Endpoint: GET /api/form-builder/forms/:formId/rules
Purpose: Retrieves all conditional logic rules defined for a given form. This is primarily for the public questionnaire rendering engine.
Logic:
Parse and validate formId from req.params.
Fetch all rules from formSchema.formRules where formId matches.
For each rule fetched, query formSchema.formRuleTargets to get its associated targets.
Structure the response to include each rule with its array of target objects.
Response:
Success (200): An array of rule objects. Each rule object should contain its properties and a nested array targets.
JSON

[
  {
    "id": 1,
    "formId": 1,
    "triggerFormPageQuestionId": 10,
    "conditionType": "equals",
    "conditionValue": "Yes",
    "actionType": "show_question",
    // ... other rule fields
    "targets": [
      { "id": 1, "ruleId": 1, "targetType": "question", "targetId": 12 },
      { "id": 2, "ruleId": 1, "targetType": "page", "targetId": 3 }
    ]
  },
  // ... more rules
]
Error (404/500): Error message.
